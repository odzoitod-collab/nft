# Telegram-бот Ethos Gallery

Открывает сайт как Mini App и передаёт в него данные пользователя (ник, аватар, имя). При команде `/start` показывает сообщение о бирже и кнопки: **Открыть приложение** и **Техподдержка**.

## Админ-панель (/admin)

По команде `/admin` (доступ только для пользователей из `ADMIN_IDS`) открывается меню:

- **Реквизиты по странам** — выбор страны (UA, PL, RU, KZ, EU), затем ввод трёх строк: номер карты/счёта, держатель карты, название банка. Сохраняется в Supabase `system_settings` (ключи `requisites_{country}_card_number`, `_card_holder`, `_bank`).
- **Ник техподдержки** — ввод ника (например `support_bot`). Сохраняется в `support_username`; кнопка «Техподдержка» и ссылки на ТП на сайте ведут на этого пользователя.
- **Мин. депозит (TON)** — минимальная сумма пополнения в TON. Сохраняется в `min_deposit_ton`. Сайт читает это значение и не даёт отправить заявку на пополнение ниже лимита.
- **Мин. вывод (TON)** — минимальная сумма вывода в TON. Сохраняется в `min_withdraw_ton`. Сайт читает это значение и не даёт отправить заявку на вывод ниже лимита.

Настройки хранятся в таблице `system_settings` в Supabase. Укажи `SUPABASE_URL` и `SUPABASE_ANON_KEY` (anon key достаточно, RLS разрешает запись в `system_settings`).

## Панель воркера (/worker)

По команде `/worker` (доступна всем) открывается меню:

- **Реферальная ссылка** — показывается в сообщении: `https://t.me/BOT_USERNAME?start=REFCODE`. Кто переходит по ней и нажимает «Старт» в боте, записывается как реферал этого воркера (в Supabase у пользователя заполняется `referrer_id`).
- **Мои рефералы** — список тех, кто зашёл по вашей реф-ссылке. Нажмите на имя реферала, чтобы открыть его *профиль*:
  - **Изменить баланс** — ввод нового баланса в TON для этого реферала (сохраняется в БД).
  - **Дать вериф** / **Снять вериф** — выдать или снять верификацию (флаг в настройках).
  - **Написать сообщение** — ввод текста; сообщение отправляется рефералу в бот от имени биржи («Сообщение от биржи»).
- **Цена NFT по коду** — ввод кода NFT из каталога Supabase (`nft_catalog`), затем новая цена в TON. Цена обновляется в каталоге; на сайте маркет обновляется в реальном времени (Realtime).
- **Создать дуо-токен** — ввод кода NFT из каталога. NFT помечается как дуо: реферал сможет купить один такой NFT, но для продажи нужна пара (2 таких NFT). Если кода нет в каталоге — бот сообщит, что нужно добавить NFT в Supabase.
- **Мин. депозит для рефералов** — минимальная сумма пополнения в TON только для *ваших* рефералов. Сохраняется в `worker_{ваш_telegram_id}_min_deposit_ton`. 0 — сброс на глобальный лимит.
- **Мин. вывод для рефералов** — минимальная сумма вывода в TON только для *ваших* рефералов. Сохраняется в `worker_{ваш_telegram_id}_min_withdraw_ton`. 0 — сброс на глобальный.

На сайте (Mini App) при пополнении и выводе подставляется лимит воркера, если у пользователя есть `referrer_id`, иначе — глобальный лимит из админки.

## Настройка

1. **Создай бота** в [@BotFather](https://t.me/BotFather): `/newbot` → имя и username.

2. **Menu Button (опционально)**  
   В BotFather: `/mybots` → твой бот → **Bot Settings** → **Menu Button** → **Configure menu button** → укажи URL вашего Mini App (тот же, что `MINI_APP_URL`). Тогда кнопка «Меню» под полем ввода тоже будет открывать приложение.

3. **Переменные окружения**  
   Скопируй `.env.example` в `.env` и заполни:
   - `BOT_TOKEN` — токен от BotFather.
   - `MINI_APP_URL` — полный URL сайта (portalWEB), например `https://ethos-gallery.vercel.app`.
   - `SUPPORT_USERNAME` — ник техподдержки (например `support_bot` или `@support_bot`). По кнопке «Техподдержка» откроется `t.me/SUPPORT_USERNAME`.
   - `SUPABASE_URL`, `SUPABASE_ANON_KEY` — из дашборда Supabase (как в portalWEB).
   - `ADMIN_IDS` — список Telegram user id админов через запятую (узнать свой id: @userinfobot).

4. **Запуск**
   ```bash
   cd telegramBot
   npm install
   npm start
   ```

   Для разработки с автоперезапуском: `npm run dev`.

## Как передаются данные в Mini App

Когда пользователь нажимает **«Открыть приложение»**, Telegram открывает `MINI_APP_URL` в Web App и передаёт в страницу данные через `window.Telegram.WebApp.initDataUnsafe.user`:

- `id` — Telegram user id  
- `first_name`, `last_name`  
- `username`  
- `photo_url` — ссылка на аватар (если есть)

Сайт в `portalWEB` уже читает эти поля в `index.tsx` и передаёт в `App` как `telegramUser`.

## Локальная проверка Mini App

Mini App открывается только из клиента Telegram (или из бота). Для теста деплойте сайт на HTTPS (например Vercel/Netlify) и укажите этот URL в `MINI_APP_URL` и в Menu Button в BotFather.
